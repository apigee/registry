Spec

A Spec describes a version of an API in a structured way. Specs provide formal descriptions that consumers can use to use a version. When specs consist of multiple files, each is a separate spec resource with its own name and a common value for the format property.

FieldTypeLabelDescription
name string

Resource name.

filename string

A possibly-hierarchical name used to refer to the file from other files.

description string

A detailed description.

create_time google.protobuf.Timestamp

Creation timestamp.

update_time google.protobuf.Timestamp

Last update timestamp.

style string

A style (format) descriptor for this spec. Possible values include "proto3", "OpenAPI-2.0", "OpenAPI-3.0". This is free-form but we expect that it will eventually be quite structured with an "API Description Format" part and possible "API Style" parts, e.g. "OpenAPI-3.0+Google" for OpenAPI v3 with (hypothetical) Google style constraints.

size_bytes int32

The size of the spec file in bytes.

hash string

A hash of the spec file's contents

source_uri string

The original source URI of the spec file (if one exists). This is an external location that can be used for reference purposes but which may not be authoritative since this external resource may change after the file is retrieved.

contents bytes

The contents of the spec file. Returned only when the FULL view is requested.

revision_id string

The revision ID of the spec. A new revision is committed whenever the spec is changed in any way. The format is an 8-character hexadecimal string.

revision_create_time google.protobuf.Timestamp

The timestamp that the revision was created.
